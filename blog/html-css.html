The best practices associated with using Classes and IDs?

Fri 20 Jan 2023

ID and class selectors are important tools to use when setting up a website, but knowing when and how to use them can often be confusing, even for experienced coders. Understanding the best practices for using these selectors can make your code easier, quicker and tidier to read and write.

Using ID and Class

ID: can be added to an element in html as id=”id-name”, and is then used as a CSS selector by adding a hash (#) before the id - #id-name {}

Class: Can be added to an element in html as class=”class-name”, and is then used as a CSS selector by adding a period (.) before the class - .class-name {}

Specificity Heirarchy

Each CSS selector is part of a styling hierarchy, the 'specificity hierarchy', this means that if an element used multiple selectors, the highest in the hierarchy will cancel the others out and will be the only one to take effect. The hierarchy is as follows:

  1. Using !important
  2. Inline-styles - using the <style> </style> in your HTML file
  3. ID
  4. Class
  5. Elements/pseudo-elements - H1 etc

This is very important for distinguishing the difference between ID and class. Look at this mushroom for example, the ID selector styles the color as red, while the class selector styles it as brown. The ID selector overrides class, so the mushroom is red.

Note: Although ID is more specific, an element can be styled with a class selector as well, as long as they are not styling the same property.

Specific vs General

You may be thinking “why would you ever use class then if ID always overrides it!?!”. Using ID selectors to style every element individually can make your code very long and messy, especially if multiple elements are styled the same way. This is where class really shines.

Due to its position in the specificity hierarchy, IDs should be thought of as unique identifiers. Multiple elements should not share the same ID.

Class selectors can apply to multiple elements and should be used as more general identifiers. Elements can have both an ID and class selector, and can actually have more than one class selector.

This saves time and helps to keep your code clean. Look at these mushrooms. Their positioning is styled through their IDs, but one has the class .pink, which colours it pink, while the other has the class .blue which colours it blue. It makes my code much cleaner by reusing the class selectors rather than typing it out again each time I want to make a new mushroom.

Summary

ID:

  • Specific
  • Elements should only have one ID, and each ID should be unique
  • Overrides class selectors
  • Used in CSS as #id-name

Class:

  • General
  • Elements can have more than one class, and classes can be reused
  • Overrides element selectors, but is overridden by ID selectors
  • Used in CSS as .class-name

Play around with Class and ID selectors until you get the hang of using them. Everyone writes code differently, but the most important thing is to stay consistent and follow the best practices to keep your code accessible and maintainable.
