JavaScript and the DOM

Fri 27 Jan 2023

What is JavaScript?

Javascript is one of the most widely used programming languages, and for good reason. Javascript can be used to add interactivity to a webpage, game, application, and almost anything you can think of. When combined with HTML and CSS, it creates a reactive and interactive webpage.

Together, Javascript, HTML and CSS form the core programming languages of front-end and web development.

You can think of it as a paintbrush, as the title font may have indicated. In web development, HTML forms the structure, the canvas. Primed with gesso, texturised, and with a plan of the painting applied, a primed canvas is where most paintings begin. In and of itself, a canvas can be appreciated - there is still a lot you can experience from a primed canvas. But how does the finished piece emerge?

Through CSS color is introduced. Color can be mixed and form created. It can also create a satisfying and beautiful piece, but without the paintbrush, movement and emotion can't be added.

With a paintbrush in hand, the artist can create a masterpiece. They can reflect life, or they can create abstract, thoughtful pieces. Art evokes emotion, and the paintbrush helps to create that experience.

Just like the paintbrush, Javascript creates movement within a website. It takes the HTML canvas and CSS medium and creates an interactive experience for users.

What is control flow?

Control flow describes the order in which code is executed. In Javascript, this generally means running from the first line of code to the last in that order. This is unless Javascript is instructed to change direction. This is most commonly experienced with loops.

Loops instruct the browser to run through the internal code a set number of times, redirecting the control flow until fully executed. They take many forms, such as the for loop, for...of, for...in, do...while etc. They may appear confusing and unintuitive at first, but they can get a little easier when you realise that many everyday actions can be thought of in terms of a loop. Take having a dinner party for example: the control flow leads you from inviting your guests in, sitting them down, interacting with them, eating with them, and then saying goodbye. But there are several loops in there somewhere that interrupt that flow as you have to complete them the specified number of times until you can move on to the next. When you're serving up dinner you have to place the food on the plate as many times as needed until you can serve everyone - if you have four guests, you repeat it four times etc.

If you want to get deeper into loops, you could also throw conditionals in there too, like an if statement. While you're going through your plating up food loop, if your guest can't eat a certain ingredient, you can skip over it, or give them something else instead, and then loop around to the next guest.

Once you begin to think of loops and conditionals as phsyical actions like this, they become a lot less intimidating.

What is the DOM, and how can I interact with it?

The DOM, or Document Object Model is a representation of a webpage, or document, as a tree-like structure, or Node Tree. Like a family tree, the node tree represents the structure of the document, its content and how each element, or nodes relate to each other. Every HTML element is its own node. Nodes are divided into a heirarchy of anscestors and descendents: grand-parent nodes, parent nodes, and child nodes.

But what's the point of all this?The answer lies at the top of the tree: every node is descended from the document. Calling the document in your code allows a user or programmer to interact with and manipulate every single node on a webpage. You can do this in Javascript by searching for the node you want to affect with: document.querySelector('ThisIsTheNodeYouAreLookingFor') for example.

Finding the node then creates endless possibilities, you could:

  • Manipulate the code immediately when the page loads by changing it, for example changing the colour of the text with document.querySelector('.class').style.color = "pink"
  • Add an Event Listener, like a cue that indicates when the code can be manipulated through actions on the webpage, for example when you click on a button it changes the color to pink: document.querySelector('button').addEventListener('click', functionThatChangesColor)

Back to the dinner party analogy, the last time though, I promise. Within your dinner party the DOM could be seen in the following way:

  • Document: The dining room
  • The Node Tree: The seating arrangement
  • The Nodes: Your guests
  • Event Listeners: Social cues

To serve dinner to a guest you must go to the dining room, walk past the other guests in the seating arrangement until you get to your chosen one, wait for a cue, like the clock striking 7pm (or a complaint if you're not good at time management), then place the food down in front of them, or: document.querySelector('guest').addEventListener('complaint', serveDinnerFunction)

Functions?

They are very useful pieces of code as they are re-usable, so you can keep calling one line every time you want to use it rather than rewriting out your code every single time.

The difference between arrays and objects

At first glance, arrays and objects appear very similar and it can be difficult to distinguish the appropriate times to use each. They both hold collections of data.

Planning for your dinner party may help (I'm sorry, I lied..... But this is the last time now).

  • Arrays: An array is like the menu that you will give your guests. You would lay out each course on your menu separately from one another, but it would all come under the title of your menu. It's the same with arrays: They are a list of elements stored under a single element. Your menu array could look like:
    Menu['Bruschetta', 'Soup', 'Fish', 'Tiramisu']
  • Objects: Objects are like a more detailed version of your list. They contain properties that represent different types of information about that object in key value pairs. They would be like the recipe for each course on your menu with different points of information describing the properties, it would written like:
    Bruschetta = {
    process: 'toasted',
    breadType: 'chiabatta',
    garlic: true,
    ingredientsCost: 13 }

As their structure is different, so is their use. They can be accessed by:

  • Arrays: Calling on a property via the index. Items in programming start from 0 rather than 1, so if someone asked about your 'bruschetta' on the menu, they would call it as: bruschetta[0].
  • Objects: Calling on a value via the key, in dot notation or bracket notation. The type of bread in your bruschetta would be called by from bruschetta.breadType , or bruschetta[breadType]
